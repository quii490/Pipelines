# QC 原理与阈值

| 状态 | 维护人 | 最后审查 |
|---|---|---|
| Draft | Pipeline maintainers | 2026-07-16 |

QC 的目标是识别数据生成或处理中的风险，并决定是否重做、排除、降级解释或继续。阈值必须对应明确的 assay、物种、数据深度和指标定义。

## 四步框架

1. **定义**：分子、分母、过滤阶段、read/fragment 单位是什么？
2. **比较**：同批样本、历史合格样本和公开 benchmark 是否一致？
3. **诊断**：输入、实验、参考资源、软件还是统计设计导致异常？
4. **处置**：保留、标记、补测、重跑或排除，并写入记录。

不要用单一阈值自动删除样本。任何排除都应保留原始 QC、理由、决策人、日期以及敏感性分析。
